Methodology

Data Sources

This dashboard integrates three registries via the Berkeley Carbon Trading Project (BCTP) database as a backbone:

  • Verra VCS — project data fetched via the Verra Registry REST API; boundaries from KML files where available
  • Gold Standard — project data from the Gold Standard Impact Registry API
  • ISO 14064-2 — 12 projects added in a single batch; all pre-validation as of the update date

Geographic Resolution

Coordinates were sourced in priority order: (1) KML polygon centroid from Verra's registry KML files, (2) API-provided centroid coordinates, (3) text-matched geocoding from project title/location fields. All project points in this dashboard represent single representative points — polygon boundaries were reduced to centroids for web performance. The geometry_source field in the detail panel records which method was used for each project.

State and District Assignment

State and district were assigned via spatial join against corrected India boundary shapefiles (EPSG:4326) using the resolved_state and resolved_district fields, built consistently across all three registries.

Data Limitations

  • Locatability: 235 of 1,892 projects (12.4%) have no usable coordinate — 234 GOLD Standard and 1 ISO project. These are shown in the unlocated projects table but do not appear on the map.
  • Coordinate quality: 25 mapped projects have coordinates outside India's boundary. 2 were corrected (axis swap, sign error); 9 are near-offshore coastal precision issues; 2 share a placeholder Bay of Bengal coordinate; 12 have genuinely incorrect coordinates from the source registry and are rendered at the registry's reported location with a warning.
  • Vintage year: Only 50.5% of projects report a first vintage year. Coverage is highly uneven — Forestry & Land Use: 7.9%, Agriculture: 11.0%, vs Renewable Energy: 67.6%. Vintage-year charts should not be used to compare category growth rates.
  • Status disagreement: The BCTP snapshot status field lags live registry status. This dashboard uses registry-current status fields (status_verra_current, status_gold_current) for all status-based displays; the snapshot field is available in downloads only.
  • Emissions estimates: For 59 projects, BCTP and registry emissions estimates diverge (major: 5, extreme: 2, not comparable: 51). No single explanation covers all cases.
  • ISO registry: 12 projects, added in a single batch, all pre-validation as of the update date, zero credits retired. This reflects registry maturity, not a data gap.
  • Shared coordinates: Many aggregated programme projects (e.g. EKI Global Cookstove Program phases) share a single coordinate reflecting a developer office or regional centroid rather than a physical project site.
